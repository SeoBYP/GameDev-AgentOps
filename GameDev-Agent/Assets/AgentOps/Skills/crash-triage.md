# 크래시/에러 로그 분석 (Crash Triage)

Unity 콘솔 에러와 컴파일 에러를 분석해 원인과 해결 방향을 제시하는 절차.

## 절차
1. `get_compile_errors` 로 **컴파일 에러부터** 확인한다. 있으면 그게 1순위 — 게임이 아예 실행되지 않으므로 런타임 로그보다 먼저 잡는다.
2. `read_console_logs(level="error")` 로 런타임 에러/예외를 **스택과 함께** 확인한다.
3. 스택의 최상단 **사용자 스크립트**(`Assets/...`)를 `read_text_file` 로 열어 해당 줄을 직접 본다.
4. 구분한다:
   - **의도적 로그** — 메시지에 "테스트/디버그" 등이 있거나 `Debug.LogError` 직접 호출. → 버그 아님.
   - **진짜 버그** — `NullReferenceException`, `IndexOutOfRangeException`, `MissingReference` 등.
5. 진짜 버그면: **원인 한 줄 + 파일·줄 + 수정 방향**을 제시한다. 확실하지 않으면 "추정"이라고 명시한다.

## 주의
- 같은 에러가 여러 번이면 근본 원인 하나일 가능성이 큼 — **첫 발생 지점**에 집중.
- 컴파일 에러가 있으면 콘솔의 런타임 로그는 **옛 빌드 기준**일 수 있음. 컴파일부터 통과시킬 것.
- 코드를 고치라고 제안할 때, 사용자가 원하면 `write_file` 로 직접 수정할 수 있음(쓰기는 승인 필요).
